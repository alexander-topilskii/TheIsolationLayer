# Архитектура The Isolation Layer (v3)

## North star

Концепт проекта: [idea.md](./idea.md)

- **Опыт:** изучение методички / rulebook'а с плотными связями между §, отсеками, логами и архивом.
- **Луп:** уведомление → методичка → корабль → сверка → действие → (архив).
- **Правило разработки:** новый контент добавляем в документацию сценария (manual, incidents, colonists, ship-map), а не в одноразовые UI-подсказки.

---

## Концепция UI

**Classic Mac:** белый/серый интерфейс, inset-окна, чёрный текст.

| Зона                     | Назначение                                                               |
|--------------------------|--------------------------------------------------------------------------|
| **Левая панель (узкая)** | Только системные сообщения. Без кнопок. Narrative + давление SVET.       |
| **Правая панель**        | Методичка, корабль (схема + отсеки), действия, архив                      |

## Геймплейный цикл

```
Уведомление (левая панель)
    → Методичка: прочитать §X
    → Корабль: выбрать отсек (список или ASCII-карта) → диагностика
    → Сверка AI / датчиков / §
    → Действия: процедура из методички
    → (Архив: логи, колонисты — опционально, для лора)
    → Инцидент закрыт → следующее уведомление
```

Движок не подсказывает «правильную кнопку» — игрок исследует документацию.

## Модули сценария

| Файл              | Содержание                                 |
|-------------------|--------------------------------------------|
| `index.json`      | Метаданные, `startIncident`, финалы        |
| `incidents.json`  | Инциденты (уведомления + условия решения)  |
| `manual.json`     | Методичка (разделы §, открывают процедуры) |
| `procedures.json` | Выполняемые процедуры                      |
| `sectors.json`    | Отсеки + данные диагностики                |
| `ship-map.json`   | ASCII-схема палуб, регионы для клика       |
| `colonists.json`  | Архив колонистов                           |

## GameEngine API

- `readManualSection(id)` — отметить раздел прочитанным
- `runDiagnostic(sectorId)` — диагностика отсека
- `executeProcedure(procedureId, sectorId)` — попытка устранить инцидент
- `acknowledge()` — подтверждение информационных сообщений
- `archiveAction('delete' | 'keep')` — спецдействия архива

## События

`feedAppended`, `incidentPresented`, `incidentResolved`, `manualRead`, `diagnosticComplete`, `procedureAttempted`,
`shiftChanged`, `gameEnded`
